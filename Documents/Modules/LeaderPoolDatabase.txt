领导牌卡池数据结构：

{ID，Name, Pic, Description, CardSet, Ability}

ID: int, 从0开始，唯一id

Name: string, 卡片名称，会被GUI显示

Pic: string, 图片文件名，因为存在同名不同卡图的卡片所以这个信息有必要存在

Description: string, 卡片实际效果描述信息，会被GUI显示

CardSet: enum CardSet {NullCSet, Northern, Nilfgaarian, Monster, Scoiatael, Neutral}  
卡牌阵营，分别对应无阵营，北方领域，尼弗迦德，怪物，松鼠党，中立卡牌。类似炉石的职业，对战中每个阵营有特殊技能，不过本文件中和此无关，无需在意。
BTW由于史凯利杰阵营更新了一堆奇怪的卡牌，那些暂时不考虑制作。

Ability: enum Ability { Pick_Rain,
						Read_Hand_Card,
						Cancel_Ability,
						Rraw_From_Opponent,
						Restore_Unit,
						Pick_Fog,
						Clear_Weathear,
						Seige_Horn,
						Seige_Scorch,
						Range_Scorch,
						Pick_Frost,
						Draw_11th,
						Range_Horn,
						Close_Scorch,
						Move_Aglie,
						Pick_Weather,
						Close_Horn,
						Destroy2_Draw1,
						Restore_Unit_ToHand,
						Double_Spies
}
领导牌能力， 类似游戏王的人物技能，限定每局游戏只能使用一次。由于每个领导牌的技能都是独特的非模板化的，实际上该数据对游戏开发来说作用有限，甚至是可以不需要的。
